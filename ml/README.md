## О разделе

**ML** — раздел, посвящённый фундаментальному изучению методов машинного обучения. В отличие от буткемпа, где акцент делался на практическом применении готовых инструментов, здесь разбирается математическая основа алгоритмов: аналитические решения, градиентный спуск, регуляризация, кросс-валидация и отбор признаков — с реализацией ключевых методов с нуля.

## Проекты

<details>
<summary><strong>ML1: Введение в машинное обучение</strong></summary>

*   **Тема:** Базовые концепции машинного обучения: типы задач, разведочный анализ данных, создание признаков и сравнение первых моделей.
*   **Задания:**
    *   **Задание 1. Типы задач ML:** классификация задач на категории (бинарная классификация, мультиклассовая классификация, регрессия, кластеризация, ранжирование); разбор реальных примеров применения ML (рекомендательные системы, антиспам, навигация, голосовые ассистенты, банковский скоринг).
    *   **Задание 2. Разведочный анализ данных (EDA):** загрузка датасета аренды квартир в формате JSON, анализ распределения целевой переменной (`price`), выявление выбросов, построение матрицы корреляций с помощью `Seaborn`.
    *   **Задание 3. Статистический анализ:** визуализация распределений с помощью гистограмм и боксплотов, удаление выбросов через перцентили (1% и 99%), анализ взаимосвязей признаков (`bathrooms`, `bedrooms`, `interest_level`) с таргетом.
    *   **Задание 4. Создание признаков:** генерация полиномиальных признаков (`PolynomialFeatures`, степень 10), анализ прироста информативности квадратичных преобразований.
    *   **Задание 5. Обучение и сравнение моделей:** обучение `LinearRegression` и `DecisionTreeRegressor` на полиномиальных признаках, реализация наивных базовых моделей (предсказание среднего и медианы), сравнение по метрикам MAE и RMSE на тренировочной и тестовой выборках.

</details>

<details>
<summary><strong>ML2: Обучение с учителем</strong></summary>

*   **Тема:** Глубокое погружение в линейные модели: вывод аналитического решения, реализация моделей с нуля через три метода оптимизации, регуляризация и нормализация признаков.
*   **Задания:**
    *   **Задание 1. Теоретические вопросы:** вывод аналитического решения линейной регрессии в матричной форме через геометрический подход (проекция вектора `y` на подпространство признаков); математическое описание влияния L1 и L2 регуляризации на решение; объяснение механизма разреживания весов в Lasso через геометрию ромба; применение нелинейных преобразований признаков (полиномиальные признаки, базисные функции).
    *   **Задание 2-3. Предобработка данных:** расширение набора признаков за счёт текстового поля `features` (извлечение топ-20 наиболее популярных удобств квартир через `Counter`), бинаризация текстовых признаков, формирование матрицы `X` из 22 признаков.
    *   **Задание 4. Кастомная линейная регрессия:** реализация класса `LinearRegressionCustom` с тремя методами обучения — аналитическим (формула нормального уравнения с `np.linalg.inv`), полным градиентным спуском (GD) и стохастическим градиентным спуском (SGD); сравнение с `sklearn.LinearRegression`.
    *   **Задание 5. Регуляризованные модели:** реализация классов `RidgeCustom` (L2, gradient: `dw += 2α·w`), `LassoCustom` (L1, gradient: `dw += α·sign(w)`) и `ElasticNetCustom` (L1+L2); сравнение с sklearn-реализациями по MAE, RMSE и R².
    *   **Задание 6-7. Нормализация признаков:** реализация `MinMaxScalerCustom` и `StandardScalerCustom` с нуля; теоретическое обоснование необходимости нормализации для градиентного спуска и регуляризации; верификация совпадения с `sklearn` (max abs diff < 1e-10).
    *   **Задание 8. Переобучение:** демонстрация катастрофического переобучения на полиномиальных признаках степени 10 (R² на тесте порядка `-10²⁸`); анализ неэффективности регуляризации при alpha=1.0 для высокой степени полинома.
    *   **Задание 9-10. Сравнение и выводы:** итоговое сравнение всех конфигураций по трём метрикам; определение лучшей модели (Lasso с MinMaxScaler / ElasticNet с StandardScaler, R² test ≈ 0.41–0.45).

</details>

<details>
<summary><strong>ML3: Валидация и отбор признаков</strong></summary>

*   **Тема:** Методы валидации моделей, стратегии разбиения данных, отбор признаков и оптимизация гиперпараметров.
*   **Задания:**
    *   **Задание 1. Теоретические вопросы:** Leave-One-Out кросс-валидация (определение, сильные стороны и ограничения); сравнение Grid Search, Randomized Search и байесовской оптимизации; методы отбора признаков (Filter, Embedded, Wrapper, Post-hoc) с разбором Pearson-корреляции, Chi2, Lasso, Permutation Importance и SHAP-значений.
    *   **Задание 2. Загрузка данных и инжиниринг признаков:** бинаризация топ-20 удобств из текстового поля `features`, MinMaxScaler-нормализация, формирование финальной матрицы признаков (22 признака).
    *   **Задание 3. Стратегии разбиения (`DataSplitter`):** реализация четырёх методов разбиения — случайный сплит на 2 части, случайный сплит на 3 части (train/val/test), датовый сплит на 2 части и датовый сплит на 3 части; детерминизм через `np.random.default_rng(seed)`.
    *   **Задание 4. Кросс-валидация (`CrossValidator`):** реализация K-Fold, Grouped K-Fold (группы целиком в одном из сплитов), Stratified K-Fold (сохранение распределения таргета) и Time Series Split (расширяющееся окно без перемешивания).
    *   **Задание 5. Сравнение схем валидации:** сравнение кастомных реализаций с `sklearn` через KS-тест распределений признаков (p=1.0 подтвердил идентичность); Time Series Split показал наилучший R² (0.58) как наиболее адекватная схема для временны́х данных.
    *   **Задание 6. Отбор признаков (`FeatureSelector`):** реализация четырёх методов отбора — по коэффициентам Lasso, по NaN-доле и матрице Pearson-корреляции, по Permutation Importance и по SHAP-значениям (`shap.LinearExplainer`); сравнение результирующих метрик и скорости работы каждого метода.
    *   **Задание 7. Оптимизация гиперпараметров (`HyperparameterOptimizer`):** реализация Grid Search (полный перебор сетки `alpha` × `l1_ratio`), Random Search (`ParameterSampler`) и байесовской оптимизации через `Optuna` (TPE-семплер) с опцией K-Fold CV; итоговое сравнение четырёх методов по времени выполнения и Val MAE.

</details>

## Как использовать

1. Клонируйте репозиторий и перейдите в директорию:
    ```bash
    git clone https://github.com/[Your-Username]/[Your-Repo-Name].git
    cd [Your-Repo-Name]/ml
    ```
2. Создайте виртуальное окружение и установите зависимости:
    ```bash
    python -m venv venv
    source venv/bin/activate  # macOS/Linux
    venv\Scripts\activate     # Windows
    pip install -r requirements.txt
    ```
